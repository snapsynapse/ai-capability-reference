# Roundup Templates

Three recurring templates for the Tuesday LinkedIn roundup post.
Select based on the week's changes (see SKILL.md Template Selection).


## Template A: changes-first

Use when 3+ platforms changed. Lead with specifics, one paragraph per
platform, tracker rides along at the end.

Structure:

- Opening: one sentence framing the scope ("Three things changed...")
- Body: one short paragraph per platform change, each with plan-level
  detail and who it affects
- Tracker mention: one sentence, framed as utility not promo
- Link
- Hashtags

Example shape:

```
Three things changed in AI tooling this week that affect what you can
actually do on each plan.

Claude: [what changed]. [plan detail]. [who it affects].

ChatGPT: [what changed]. [plan detail]. [who it affects].

Gemini: [what changed]. [plan detail]. [who it affects].

I track these weekly across six platforms so trainers and operators
don't get burned mid-session by stale info.

https://snapsynapse.github.io/ai-capability-reference

#AIStrategy #FutureOfWork
```

Target: 900-1,200 characters.


## Template B: tension opener

Use when the changes reveal a pattern or trend worth naming. Lead with
the pattern (abstract), then reveal the specifics. Stronger above-the-fold
pull than Template A.

Structure:

- Opening: 2 lines that name the tension without naming platforms
  (must work as standalone preview)
- Body: the specific changes, framed as evidence of the pattern
- Frame: one sentence naming what the pattern means for the reader
- Tracker mention: framed as "the memo"
- Link
- Hashtags

Example shape:

```
The hardest part of teaching AI tools isn't explaining what they do.
It's knowing what changed since last Tuesday.

This week across the six major platforms:

[Change 1, abstracted]. [Change 2, abstracted]. [Change 3, abstracted].

These aren't blog-post-worthy announcements. They're the kind of
changes that make a trainer look wrong in front of a room.

I publish a weekly tracker so the memo exists in one place.

https://snapsynapse.github.io/ai-capability-reference

#AIStrategy #FutureOfWork
```

Target: 900-1,300 characters.


## Template C: shortest viable

Use for light weeks (1-2 small changes) or low-energy weeks. Just the
signal, no narrative. Keeps the streak alive without forcing content.

Structure:

- Opening: one line framing ("AI feature changes worth knowing...")
- Body: bullet-style lines, one per change, with plan and price detail
- Tracker mention: one sentence
- Link
- Hashtags

Example shape:

```
AI feature changes worth knowing this week:

ChatGPT Deep Research: now on Plus ($20/mo), was Pro-only. 5/month limit.
Claude Cowork: GA on macOS. No Windows or Linux yet.

I track plan-by-plan availability across six platforms weekly.

https://snapsynapse.github.io/ai-capability-reference

#AIStrategy #FutureOfWork
```

Target: 600-900 characters.


### Quiet-week variant

When no changes were detected across any tracked platform:

```
No major feature or plan changes across the six platforms I track
this week. That's worth noting too: stability is data.

If you're prepping sessions or making buying decisions, everything
in the reference is current as of [date].

https://snapsynapse.github.io/ai-capability-reference

#AIStrategy #FutureOfWork
```

Target: under 600 characters. Do not pad a quiet week with filler.
