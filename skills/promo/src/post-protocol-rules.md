# Post Protocol Rules (Roundup Extract)

This file captures the subset of Post Protocol and LinkedIn voice rules
that apply to the Tuesday roundup. It exists so the weekly-roundup skill
can be self-contained without requiring the full Post Protocol or
linkedin-post skill to be loaded.

Source documents (authoritative, consult if conflicts arise):

- Post Protocol 2026Q1 (LocalBrain: 8_Reference/SOPs/Post Protocol 2026Q1.md)
- LinkedIn Voice Profile (linkedin-post skill: references/linkedin-voice.md)


## Schedule

Tuesday 10 AM Mountain Time.
Manual posting from mobile preferred.


## Lane

Bridge: AI onramps for humans, practical enablement, measurement over vibes.


## Packaging Order

Every post follows this sequence where the content supports it:

1. Expensive failure (what goes wrong when this info is stale)
2. Human behavior under pressure (the trainer mid-session, the buyer mid-decision)
3. Instrument (what to verify or measure: the tracker)
4. Smallest fix (bookmark it, check before your next session)

Not every roundup hits all four. Steps 3 and 4 are always present.


## Formatting

- Plaintext only. LinkedIn does not render markdown.
- No bold. No italics.
- No em dashes. Use commas, periods, or colons.
- No emoji.
- Line breaks are structure. One sentence per line for openings.
- Short paragraphs: 1-3 sentences each.
- One idea per paragraph.


## Character Targets

- Standard roundup: 600-1,300 characters
- Quiet week: under 600 characters


## Links

Link at the end of the post, inline. Never in comments.


## Hashtags

0-2 maximum. Placed at end.

Approved lanes for the roundup:

- #AIStrategy (primary)
- #FutureOfWork (primary)
- #AIAdoption (situational, when changes affect rollout decisions)

Avoid: #AI (too broad), #AIskills (too niche), #Leadership (generic)


## One-Ask Rule

Exactly one CTA per post. For roundups, the CTA is implicit: the
link itself. Do not add "What changes surprised you?" or "Thoughts?"
unless the week's content genuinely invites a specific, answerable
question.


## Spice Level

3/10. External-calm. Direct claims, constructive tone.
The roundup is informational, not editorial.


## Anti-Patterns

Hard stops for roundup posts:

- "I'm excited to share..."
- "Here are N things..."
- "Game-changer / revolutionize / disrupt"
- "Thoughts?" as CTA
- Describing the tracker before describing the changes
- Making the post about the build process or the tracker itself
- Padding a thin week with filler or commentary
- Including unverified changes (everything must trace to repo data)


## Measurement

Track per the Post Protocol weekly review:

- Impressions
- Actions per impression
- Saves per impression (roundups should index on saves)
- Comments per impression
- Profile clicks per impression

The roundup is designed for saves. If saves rate is consistently low,
the changes reported are not specific enough or the plan-level detail
is missing.
